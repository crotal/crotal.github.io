Write Posts

Create Posts

You can create a new post by using the following command or just create it by yourself.

$ crotal new_post 'hello-world'

The Header Area

In a post file, there is a default header area like this:

---
layout: post
title: "Hello, World"
date: 2013-04-20 15:10
categories: crotal,docs
slug: hello-world
---

This is written in yaml, the following table explains what each of them does:

Name Description
layout Layout indicates the template you prefer to use to generate the post, see Templates for more information.
title The post's title.
date Date the post were generated, when generating, it will be turned into a datetime object.
categories The categories the post belonging to.
slug Slug is a short string describing the post, when generating, slug will be turned into the url of the post.See Permalinks for more information.
comment

However, there are some more infos which were not included in the default header area.

Name Description
comments When comments was set to true, comment block will be shown at the bottom of the post, when set to false, it will now shown up.
description Description stand for a short description of the post, when people found your post from search engine, they will firstly read the description from your post, if it is not set, Crotal will use the first 255 characters of your post to generate a auto description.

The Main Body of Posts

When writing the main body of the post, you should use markdown syntax, it is really easy, see Markdown Syntax for more information.

HTML is also suported

You can use the combination of HTML and markdown in one post.

How to Include Images

Before including any images, you should place it in the static/ folder like this:

▾ static/
    ▾ images/
        foo.png

Then you can include the image by using the syntax as followed:

![Image Description](/images/foo.png)

How to Include Code Blocks

Including a code block is really easy, you need to write your code like this:

```Language
Code Line 1
Code Line 2
Code Line 3
Code Line 4
Code Line 5
```

language indicates the language you want to include, if you didn't set it, Crotal will automaticaly indentify the language, which is not always accurate.

For example, including a piece of python code should be like this:

```Python
from pages import Page
from posts import Post

class Category():

    def __init__(self, config, category):
        self.title = category
        self.posts = []
```

Here is the result:

from pages import Page
from posts import Post

class Category():

    def __init__(self, config, category):
        self.title = category
        self.posts = []

It's that easy.
